# A Sense of Engineering

어엿한 엔지니어라면 좋은 미감(美感, 여기서는 sense...)을 가져야 한다고 믿습니다.

어떤 코드와 시스템, 의사결정을 보면서, "구리다", "아름답다". "별론데 이해는 간다" 따위의 심미안을 가지고 판단할수 있어야 한다고 믿습니다.

매 챕터마다 나름의 주제를 던지고 제가 생각하는 미감 포인트를 공유드리려고 합니다.

'실전 팁'과는 거리가 조금 있구요. '교과서만 봤는데 시험을 잘 봤어요' 의 '교과서'와 좀 더 가까운 이야기들입니다.

예를 들면, 아래와 같은 주제들을 다루려고 합니다.

- 개발자는 기획자가 시킨것만 정확히 만들면 될까요?
- 테스트가 뭐에요 대체? 왜 짜요? 어떻게 짜요? 왜요...?
- 언제 return 하고, 언제 raise 하면 되나요?
- 그래서 DB 뭐쓰면 되죠?
- 리뷰 해보고 싶은데, 어떻게 해요?
- 난생 처음 보는 기술 스택 어떻게 접근해요?
- 거의 비슷한데 조금 다른 비즈니스 로직 짜야하는데... 복붙할까요? 상속할까요?

특정 언어라던가, 기술적 디테일과는 거리가 먼 포괄적인 논제들을 다루려고합니다.

디테일한 예시들이 등장하나 이는 예시일 뿐입니다.

도슨트가 소개해주는 미술과 같습니다.

예시에서 아름다움을 느껴보셨으면 좋겠지만, 꼭 느끼지 않아도 괜찮습니다. 필자의 의견일 뿐이니까요.

대신 예시들 사이를 거니며 본인만의 관점과 미감을 찾아보세요.
