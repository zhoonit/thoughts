# A Sense of Engineering

어엿한 엔지니어라면 어떤 코드와 시스템, 의사결정을 보면서, "구리다", "아름답다". "별론데 이해는 간다" 따위의 감정을 가질수 있어야 한다고 믿습니다.

매 챕터마다 나름의 주제를 던지고 제가 생각하는 미감 포인트를 공유드리려고 합니다.

포괄적인 논제들을 다루려고하고, 엔지니어다운 엔지니어로 성장하는데 도움될만한 이야기를 목표로 합니다.

('실전 팁'과는 거리가 조금 있구요. '교과서만 봤는데 시험을 잘 봤어요' 의 '교과서'와 좀 더 가까운 이야기들입니다.)

예를 들면, 아래와 같은 주제들을 다루려고 합니다.

- 개발자는 기획자가 시킨것만 정확히 만들면 될까요?
- 테스트가 뭐에요 대체? 왜 짜요? 어떻게 짜요? 왜요...?
- 언제 return 하고, 언제 raise 하면 되나요?
- 그래서 DB 뭐쓰면 되죠?
- 리뷰 해보고 싶은데, 어떻게 해요?
- 난생 처음 보는 기술 스택 어떻게 접근해요?
- 거의 비슷한데 조금 다른 비즈니스 로직 짜야하는데... 복붙할까요? 상속할까요?
