# Why Engineering Matters

Good engineering is about making deliberate choices — understanding trade-offs, not just following patterns.

<Callout type="info" title="What this chapter covers">
We'll explore why engineering discipline matters and how small decisions compound over time.
</Callout>

## Code is Communication

Code is read far more often than it is written. Clarity is not optional.

<CodeBlock language="typescript">
{`// Clever but unclear
const data = arr.reduce((a, b) => ({...a, [b.id]: b}), {});

// Clear and intentional
const map = new Map(arr.map(item => [item.id, item]));`}
</CodeBlock>

<Callout type="tip" title="Rule of thumb">
If you need a comment to explain *what* the code does (not *why*), the code should be rewritten.
</Callout>
